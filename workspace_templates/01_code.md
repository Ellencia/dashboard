# 프로젝트 문서 작성 원칙 — STATUS.md / update.md

이 문서는 **모든 프로젝트가 공통으로 갖춰야 하는 두 개의 마크다운 파일**의 작성 양식을 정한다.
AI(작업 도우미)에게 프로젝트 작업을 시킬 때, 이 문서를 그대로 지시 기준으로 삼는다.

---

## 0. 개요

- 각 프로젝트 폴더의 **최상위**에 `STATUS.md` 와 `update.md` 두 파일을 둔다.
- 이 두 파일은 **프로젝트 대시보드 위젯**이 자동으로 읽어
  진행률 · 할 일 · 최근 변경을 바탕화면에 표시한다.
- 따라서 두 파일은 **사람이 읽는 문서이자, 프로그램이 파싱하는 데이터**다.
  아래 양식을 정확히 지켜야 위젯이 올바르게 인식한다.

| 파일 | 역할 | 위젯에 표시되는 것 |
|---|---|---|
| `STATUS.md` | 진행 상황 (체크리스트) | 프로젝트 이름, 진행률 %, 메모, 할 일 목록 |
| `update.md` | 변경 이력 (날짜별 기록) | `🕒 최근 변경` 줄 (가장 최근 날짜 + 요약) |

---

## 1. STATUS.md — 진행 상황

### 양식

```markdown
# 프로젝트 표시 이름

> 한 줄 메모 (선택)

- [x] 이미 끝낸 작업
- [x] 또 끝낸 작업
- [ ] 아직 남은 작업
- [ ] 또 남은 작업
```

### 규칙

1. **파일명은 정확히 `STATUS.md`**, 위치는 프로젝트 폴더 최상위.
2. 맨 위 **첫 번째 `# 제목`** → 위젯에 표시될 프로젝트 이름. (없으면 폴더 이름이 쓰임)
3. 첫 번째 **`> 인용` 줄** → 위젯에 작게 뜨는 한 줄 메모. (선택)
4. 체크박스 줄 형식 — 반드시 다음을 지킬 것:
   - `- [ ] 내용` = 미완료 / `- [x] 내용` = 완료 (`-` 대신 `*` 도 가능)
   - `-`와 `[` 사이 **공백 1칸**, 대괄호 **안에 공백 또는 x 한 글자**, `]` 뒤 **공백 1칸** 필수.
   - 잘못된 예: `-[ ]내용` , `- []내용` , `- [v] 내용` , `- [완료] 내용` → 모두 인식 안 됨.
5. **진행률(%) = 체크된 항목 수 ÷ 전체 체크박스 수** (자동 계산).
6. 미완료(`- [ ]`) 항목은 위젯의 **"할 일" 목록**에 모인다.
7. 항목은 **마일스톤 · 기능 단위**로 적는다. 너무 잘게 쪼개지 말 것 (5~12개 권장).
8. 체크박스가 아닌 설명·소제목은 자유롭게 넣어도 되지만, 진행률 계산엔 영향을 주지 않는다.

## 2. update.md — 변경 이력

### 양식

```markdown
# 변경 이력

## 2026-05-21
- 가장 최근에 한 일 (이 첫 줄이 위젯 카드에 표시됨)
- 그 외 그날 한 일

## 2026-05-20
- 그 전에 한 일
```

### 규칙

1. **파일명은 정확히 `update.md`**, 위치는 프로젝트 폴더 최상위.
2. 변경 한 묶음 = **`## 날짜` 블록 하나**.
   - 날짜 형식: `2026-05-21` (YYYY-MM-DD 권장. `2026.05.21`, `2026/05/21` 도 인식됨)
3. **항상 최신 날짜를 맨 위에** 적는다 (역순 정렬).
   새 변경이 생기면 파일 맨 위에 `## 새 날짜` 블록을 추가한다.
4. 각 날짜 블록 안에는 **`- 내용`** 글머리표로 그날 한 일을 적는다.
5. 위젯 카드의 `🕒 최근 변경` = **날짜가 들어간 첫 번째 `## 제목`** + **그 블록의 첫 줄**.
   → 그 날의 가장 중요한 변경을 **첫 줄**에 둘 것.
6. **날짜 없는 `## 제목`**(예: `## Current State`, `## 개요`)은 최근 변경으로 잡히지 않는다.
   요약 섹션 등을 둬도 무방하지만, 위젯엔 날짜 블록만 반영된다.
7. (선택) 제목에 설명을 붙이는 형식도 허용: `## 2026-05-21 — 검색 정확도 개선`.
   이 경우 `—` 뒤 설명이 카드에 우선 표시된다.

## 3. 공통 규칙

1. **인코딩은 반드시 UTF-8** (한글 깨짐 방지).
2. 두 파일 모두 **프로젝트 폴더 최상위**에 위치.
3. 위젯은 **작업 폴더(이 파일이 있는 폴더) 아래**(하위 폴더 포함, 기본 4단계까지)에서
   STATUS.md가 있는 폴더를 **자동으로** 찾는다. 새 프로젝트는 이 안 어디에든
   폴더를 만들고 STATUS.md만 넣으면 별도 등록 없이 위젯에 나타난다.
4. 새 프로젝트의 두 파일은 대시보드 폴더의 `STATUS_TEMPLATE.md`·`UPDATE_TEMPLATE.md`
   를 복사해 쓰거나, 위젯에서 GUI로 만들 수 있다
   (제목줄 우클릭 → "새 프로젝트 만들기", 카드 우클릭 → "편집").

## 4. AI 지시문 (복사해서 사용)

> 프로젝트 작업을 AI에게 시킬 때, 아래 문단을 그대로 붙여넣어 지시한다.
> (Claude Code·Codex·Gemini CLI 는 작업 폴더의 CLAUDE.md·AGENTS.md·GEMINI.md
>  를 자동으로 읽으므로 따로 붙여넣지 않아도 됨.)

```text
이 프로젝트 폴더 최상위에 STATUS.md 와 update.md 두 파일을 항상 유지·갱신할 것.

[STATUS.md]
- 첫 줄에 "# 프로젝트 이름", 그 아래 "> 한 줄 메모"(선택).
- 작업 항목을 "- [ ] 내용"(미완료) / "- [x] 내용"(완료) 체크리스트로 적을 것.
  ('-'와 '[' 사이 공백, 대괄호 안 공백/ x 한 글자, ']' 뒤 공백 필수)
- 항목은 마일스톤·기능 단위 5~12개. 작업이 끝나면 해당 항목을 [x]로 바꿀 것.

[update.md]
- 첫 줄 "# 변경 이력". 그 아래 변경 묶음을 "## YYYY-MM-DD" 블록으로 적을 것.
- 항상 최신 날짜 블록을 파일 맨 위에 추가(역순). 각 블록 안은 "- 내용" 글머리표.
- 그날 가장 중요한 변경을 블록의 첫 줄에 둘 것.
- 작업을 수행했으면 오늘 날짜 블록에 한 줄 이상 반드시 기록할 것.

[공통]
- 두 파일 모두 UTF-8로 저장. 파일명은 정확히 STATUS.md / update.md.
- 기존 내용을 함부로 지우지 말고, STATUS는 체크 상태만 갱신, update는 위에 덧붙일 것.
```
