# 프로젝트 대시보드

내 프로젝트들의 진행 상황과 할 일을 바탕화면 위에 항상 띄워두는 작은 위젯.
포스트잇처럼 화면 구석에 떠 있고, 각 프로젝트의 `STATUS.md`(진행 상황)·`update.md`(변경 이력)를 읽어 자동으로 갱신됨.

```
┌─ 📌 프로젝트 보드 ──── ↻ — ✕ ┐
│ 프로젝트 3개 · 남은 할 일 12개  │
│ ┌────────────────────────┐ │
│ │ PPS · 공공조달 자문 RAG  73% │ │
│ │ ███████████████░░░░░░       │ │
│ │ 완료 8 / 11                 │ │
│ │ 🕒 2026-05-18 · 최근 변경     │ │
│ └────────────────────────┘ │
│ 📋 할 일   12                  │
│ ☐ 답변 캐싱 (자주 묻는 질문) PPS │
│ ☐ 도면 비교 / 검수    MCP-cad   │
│ 업데이트 14:30:21              │
└────────────────────────────┘
```

---

## 1. 필요한 것

- **Python 3.10 이상** (확인: PowerShell에서 `python --version`)
- **위젯 모드**: 추가 설치 패키지 없음 (파이썬 기본 `tkinter`만 사용)
- **트레이 모드**: `pip install pystray pillow` 필요

---

## 2. 실행 방법

| 파일 | 용도 |
|---|---|
| `run.bat` | 평소 실행 (콘솔 창 없이 위젯만 뜸) |
| `run_debug.bat` | 문제 생겼을 때 — 오류 메시지를 콘솔에 보여줌 |
| `install_autostart.bat` | 윈도우 켤 때 자동 실행되도록 등록 |
| `uninstall_autostart.bat` | 자동 실행 등록 해제 |

**처음엔 `run_debug.bat`으로 실행**해서 오류가 없는지 확인한 뒤,
이상 없으면 평소엔 `run.bat`을 쓰면 됨.

위젯이 마음에 들면 `install_autostart.bat`을 한 번 실행 →
이후 컴퓨터를 켤 때마다 자동으로 떠 있음.

## 3. 위젯 조작법

| 동작 | 방법 |
|---|---|
| 창 옮기기 | 위쪽 제목 표시줄(글자 부분)을 마우스로 끌기 |
| 항상 위 고정 켜기/끄기 | 제목줄 왼쪽의 `📌` 버튼 (파란색=켜짐, 회색=꺼짐) |
| 새로고침 | 제목줄 오른쪽의 `↻` 버튼 |
| 설정 창 열기 | 제목줄 오른쪽의 `⚙` 버튼 |
| 접기 / 펴기 | 제목줄 오른쪽의 `—` 버튼 |
| 닫기 | 제목줄 오른쪽의 `✕` 버튼 |
| 메뉴 열기 | 제목 표시줄에서 **마우스 우클릭** |
| 프로젝트 편집 | 카드 **우클릭 → "편집..."** — 할 일·메모·이름·변경기록을 GUI로 수정 |
| 새 프로젝트 만들기 | 제목 표시줄 **우클릭 → "새 프로젝트 만들기..."** |
| 할 일 완료 체크 | 할 일 줄을 클릭 (다시 클릭하면 해제) |
| STATUS.md 열기 | 프로젝트 이름을 클릭 |
| update.md 열기 | 카드의 `🕒 최근 변경` 줄을 클릭 |
| 카드 접기 / 펴기 | 카드 왼쪽 `▾` / `▸` 화살표 클릭 (우클릭 메뉴에도). 접으면 그 프로젝트 할 일도 목록에서 빠짐 |
| 할 일 목록 접기 / 펴기 | `📋 할 일` 헤더를 클릭 (전체 목록을 접음) |
| 보드 접기 / 펴기 (단축키) | 전역 단축키 — 기본 `ctrl+alt+d` (설정 창에서 변경, 어느 창에서든 작동) |
| 프로젝트 숨기기 | 프로젝트 카드에서 **우클릭** → "숨기기" |
| 숨긴 프로젝트 복원 | 맨 아래 `▸ 숨김 N개` 클릭 → 펼친 뒤 프로젝트 줄 클릭 |

> 각 버튼에 마우스를 올리면 설명 풍선이 뜸.
> 위젯은 일반 창이 아니라서 작업표시줄에 안 나타남. 닫을 땐 `✕` 버튼을 쓸 것.
> 닫은 뒤 다시 띄우려면 `run.bat`을 실행하면 됨.

## 4. 진행 상황은 어떻게 표시되나 — `STATUS.md`

각 프로젝트 폴더에 **`STATUS.md`** 파일을 두면 위젯이 자동으로 인식함.
형식은 간단한 마크다운 체크리스트:

```markdown
# PPS · 공공조달 자문 RAG

> 한 줄 메모 (선택)

- [x] 끝낸 작업
- [x] 또 끝낸 작업
- [ ] 아직 남은 할 일
- [ ] 또 남은 할 일
```

- 맨 위 **`# 제목`** → 위젯에 표시될 프로젝트 이름
- 첫 번째 **`> 인용`** 줄 → 위젯에 작게 뜨는 메모
- **`- [x]`** = 완료, **`- [ ]`** = 미완료
- **진행률(%) = 체크된 항목 수 ÷ 전체 항목 수**
- 미완료 `- [ ]` 항목들이 위젯 아래쪽 **"할 일"** 목록에 모임

새 프로젝트는 `C:\Programming_STC` 아래 **어디에든**(하위 폴더 포함) 폴더를
만들고 그 안에 `STATUS.md`를 넣으면 **자동으로** 위젯에 나타남 —
`config.json`에 따로 등록할 필요 없음. `STATUS_TEMPLATE.md`를 복사해 쓰면 편함.

### 할 일 추가·수정 — GUI 편집 창 (권장)
프로젝트 **카드를 우클릭 → "편집..."** 하면 편집 창이 열림:
- 할 일 추가 · 이름 변경 · 삭제 · 완료 체크
- 프로젝트 이름 · 메모 수정
- `update.md` 변경 기록 한 줄 추가 (날짜별로 자동 정리)

텍스트 파일을 직접 안 고쳐도 됨. 물론 `STATUS.md`를 열어 `- [ ] 내용` 줄을
직접 적어도 되고, 저장하면 다음 새로고침(기본 30초) 때 위젯에 반영됨.

## 5. 변경 이력 — `update.md` (선택)

프로젝트 폴더에 **`update.md`** 를 두면, 위젯 카드에 **`🕒 최근 변경`** 줄이
함께 표시됨. 형식은 날짜별 마크다운 목록:

```markdown
# 변경 이력

## 2026-05-20
- 가장 최근에 한 일 (이 줄이 카드에 표시됨)
- 그 외 그날 한 일

## 2026-05-19
- 그 전에 한 일
```

- 날짜가 들어간 **첫 번째 `## 제목`** 을 최신 항목으로 봄 → **최신을 맨 위에** 적을 것
- 카드에 뜨는 문구 = **가장 최근 날짜 + 그 블록의 첫 줄**
- 카드의 `🕒 최근 변경` 줄을 클릭하면 `update.md` 가 열림
- `update.md` 가 없는 프로젝트는 이 줄이 그냥 안 뜸 (있어도 없어도 무방)
- `## Current State` 처럼 날짜 없는 제목은 최근 변경으로 잡히지 않음 (PPS 형식 호환)
- 새 프로젝트엔 `UPDATE_TEMPLATE.md` 를 복사해 `update.md` 로 쓰면 됨

## 6. AI가 STATUS.md·update.md를 자동으로 쓰게 하기

이 위젯은 각 프로젝트의 `STATUS.md`·`update.md`만 읽어 표시함. 즉 위젯이 늘
최신 상태이려면 **프로젝트 작업을 할 때마다 그 두 파일이 갱신돼야** 함.
이걸 매번 손으로 하지 않고 **AI에게 맡기는** 방법:

### 이미 자동화돼 있음 — Claude Code / Codex / Gemini CLI

워크스페이스 루트 `C:\Programming_STC` 에 `CLAUDE.md`·`AGENTS.md`·`GEMINI.md`
세 파일이 있고, 각 AI 도구가 세션 시작 시 자동으로 읽음. 세 파일 모두
**"작업하는 프로젝트의 STATUS.md·update.md를 `01_code.md` 규격대로 유지하라"**
는 지시를 담고 있음 → 이 세 도구로 작업하면 **따로 지시하지 않아도** 두 파일이
자동으로 관리됨.

작성 규격의 단일 출처는 **`C:\Programming_STC\01_code.md`** — 규칙을 바꾸려면
그 파일만 고치면 모든 AI에 반영됨.

### 그 외 AI (웹 ChatGPT·Gemini 등)

파일을 자동으로 못 읽으므로, 작업을 시킬 때 아래 지시문을 붙여넣으면 됨:

```text
지금 작업하는 프로젝트 폴더 최상위에 STATUS.md(진행 상황)와 update.md(변경 이력)를
유지한다. 바탕화면 대시보드 위젯이 이 두 파일을 읽어 진행률·할 일·최근 변경을 보여준다.
작업을 하면 항상:
 - STATUS.md: 작업 항목을 "- [ ]"(미완료) / "- [x]"(완료) 체크리스트로 갱신
 - update.md: 맨 위에 "## YYYY-MM-DD" 블록으로 변경 내용을 한 줄 이상 기록
두 파일 모두 UTF-8로 저장. 자세한 양식은 C:\Programming_STC\01_code.md 를 따른다.
```

### 새 프로젝트를 시작할 때

폴더를 `C:\Programming_STC` 아래 어디든 만들고, AI에게 한 번만 시키면 됨:

> "이 프로젝트에 STATUS.md·update.md를 01_code.md 규격대로 만들어줘"

`STATUS.md`만 들어가면 위젯이 자동으로 잡아 표시함 (`config.json`의 `auto_discover`).

## 7. 설정 — `config.json`

| 항목 | 설명 | 기본값 |
|---|---|---|
| `root` | 프로젝트들을 찾을 최상위 폴더 | `c:/Programming_STC` |
| `projects` | 추적할 폴더를 직접 지정 (root 밖이거나 강제 포함용. 보통 비워둠) | `[]` |
| `auto_discover` | `root` 아래를 **재귀적으로** 훑어 `STATUS.md` 폴더 자동 탐색 | `true` |
| `scan_depth` | 자동 탐색 시 `root`에서 몇 단계까지 내려갈지 | `4` |
| `exclude` | 자동 탐색에서 제외할 폴더 이름 목록 (문서 폴더 등) | `[]` |
| `hidden` | 위젯에서 숨긴 프로젝트 폴더 이름 목록 (우클릭으로 자동 갱신) | `[]` |
| `collapsed` | 위젯에서 카드를 접어 둔 프로젝트 폴더 이름 목록 (자동 갱신) | `[]` |
| `refresh_seconds` | 몇 초마다 파일을 다시 읽을지 | `30` |
| `display_mode` | 표시 방식 (`widget` / `wallpaper` / `tray`) | `widget` |
| `widget.x`, `widget.y` | 위젯 창 위치 (px) — 닫을 때 자동 저장됨 | `60`, `60` |
| `widget.width` | 위젯 창 너비 (px) | `340` |
| `widget.opacity` | 투명도 (0.0~1.0) | `0.96` |
| `widget.theme` | `dark` 또는 `light` | `dark` |
| `widget.topmost` | 항상 다른 창 위에 표시할지 | `true` |
| `widget.max_todos` | 할 일 목록에 한 번에 보일 최대 개수 | `12` |
| `widget.todos_collapsed` | 할 일 목록을 접어 뒀는지 (헤더 클릭으로 자동 갱신) | `false` |
| `widget.collapse_highlight` | 접었을 때 제목 막대를 눈에 띄는 색으로 | `true` |
| `widget.collapse_hotkey` | 보드 접기/펴기 전역 단축키 (빈 값이면 끔) | `"ctrl+alt+d"` |

대부분의 설정은 헤더 **`⚙` 버튼 → 설정 창**에서 GUI로 바꿀 수 있음 (저장하면 새 설정으로
위젯이 다시 시작됨). `config.json`을 직접 고친 경우엔 위젯을 닫았다가 다시 실행하면 적용됨.

> **위젯이 화면 밖으로 나가 안 보일 때**: `config.json`을 열어
> `widget`의 `x`, `y`를 `60`, `60` 정도로 고치고 다시 실행.

## 8. 파일 구성

```
dashboard/
├── core.py              # STATUS.md·update.md 읽기·진행률 계산 (표시 모드 공용)
├── widget.py            # 위젯 모드 화면 (항상 위 창)
├── tray.py              # 트레이 모드 (트레이 아이콘 + 위젯 팝업)
├── hotkey.py            # 전역 단축키 (Windows RegisterHotKey)
├── editor.py            # 프로젝트 편집 창 · 새 프로젝트 만들기 창
├── main.py              # 진입점 — config의 display_mode로 모드 선택
├── config.json          # 설정 파일
├── STATUS.md            # 이 대시보드 자신의 진행 상황
├── update.md            # 이 대시보드 자신의 변경 이력
├── STATUS_TEMPLATE.md   # 새 프로젝트에 복사해 쓸 STATUS.md 본
├── UPDATE_TEMPLATE.md   # 새 프로젝트에 복사해 쓸 update.md 본
├── run.bat              # 평소 실행
├── run_debug.bat        # 오류 확인용 실행
├── install_autostart.bat   # 시작프로그램 등록
└── uninstall_autostart.bat # 시작프로그램 해제
```

## 9. 표시 모드 — `display_mode`

`config.json`의 `display_mode` 값으로 표시 방식을 고름:

| 값 | 설명 |
|---|---|
| `widget` | 항상 위에 떠 있는 위젯 창 |
| `tray` | 작업표시줄 트레이 아이콘 — 클릭하면 위젯 팝업 |
| `wallpaper` | (준비 중) 바탕화면 배경에 합성 |

모드를 바꾼 뒤 위젯을 닫았다가 다시 실행하면 적용됨.

### 트레이 모드 사용법
- `pip install pystray pillow` 가 필요함 (안 깔려 있으면 위젯 모드로 대체 실행)
- 시작하면 위젯은 안 보이고 트레이 아이콘만 생김
- 트레이 아이콘 **클릭** → 위젯 팝업이 뜸
- 팝업의 **✕** → 종료가 아니라 트레이로 다시 숨김
- 트레이 아이콘 **우클릭** → 열기 / 숨기기 / 종료
- 아이콘 색 = 전체 진행률 (분홍 → 노랑 → 파랑 → 초록)

> 위젯 모드로 되돌리려면 `config.json`의 `display_mode`를 `"widget"`으로 바꾸면 됨.

## 10. 문제 해결

| 증상 | 원인 / 해결 |
|---|---|
| 위젯이 안 뜸 | `run_debug.bat`으로 실행해 오류 확인. 같은 폴더 `error.log`도 확인 |
| "프로젝트를 찾지 못함" 표시 | 추적할 폴더에 `STATUS.md`가 있는지, `config.json`의 `root` 경로가 맞는지 확인 |
| 진행률이 0% | `STATUS.md`의 체크박스가 `- [ ]` / `- [x]` 형식인지 확인 (`-`와 `[` 사이 공백, 대괄호 안 공백 필수) |
| 한글이 깨짐 | `STATUS.md`, `config.json`을 **UTF-8**로 저장했는지 확인 |
| 위젯이 화면 밖으로 사라짐 | `config.json`의 `widget.x`, `widget.y`를 작은 값으로 수정 |
| 자동 실행이 안 됨 | `install_autostart.bat`을 다시 실행. 시작 폴더는 `Win+R` → `shell:startup` |
